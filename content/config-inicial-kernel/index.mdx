Configurar um ambiente de desenvolvimento para o Kernel Linux na disciplina MAC0470 não foi uma expêriencia muito tranquila, mas com certeza trouxe muito aprendizado. Inicialmente, seguindo cuidadosamente os tutoriais do FLUSP, me deparei com uma série de obstáculos inesperados.

### Problemas de Espaço e Compatibilidade

A primeira grande barreira surgiu já no setup inicial. Ao tentar instalar os componentes necessários, recebi uma notificação de pouco espaço disponível. Isso me obrigou a redimensionar a partição da `home`, o que pensei que resolveria todos os problemas. No entanto, logo após, enfrentei o maior desafio: uma incompatibilidade de versões que não conseguia resolver de maneira alguma.

### Apoio e Soluções

Depois de consultar amigos e até o professor, descobri que o problema era mais profundo. Fazia dois anos que eu possuía minha máquina e mais de um ano que não realizava atualizações. O setup do meu notebook estava uma verdadeira gambiarra com várias instalações estranhas. A solução foi deletar esses programas problemáticos e finalmente conseguir atualizar o sistema.

### Reflexões e Continuidade

Usar a mesma máquina para trabalho e desenvolvimento transformou a situação em um caos, mas depois de muito esforço, tudo foi resolvido, permitindo-me prosseguir com o setup. Para evitar esse tipo de problema de novo que acabou tirando muitas horas e rendendo mutio estresse, agora os dois comandos abaixos já estão fixos na minha cabeça e aprendi que devo me policiar para fazẽ-los com certa frequência

```bash
sudo apt-get update  # Atualizar pacotes
sudo apt-get upgrade  # Upgrade de pacotes

```
Agora, com um ambiente de desenvolvimento estável, estou ansioso para fazer meu primeiro commit no Kernel e realmente começar a contribuir com o desenvolvimento de software livre.
