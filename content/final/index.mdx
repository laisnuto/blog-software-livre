A disciplina de MAC0470 foi uma jornada intensa e enriquecedora no mundo do Software Livre. Desde o início, fomos introduzidos a conceitos fundamentais, passando por contribuições práticas e enfrentando desafios reais. Neste post, vou compartilhar os principais aprendizados e experiências ao longo do semestre.

### Kernel

No início da disciplina, seguimos tutoriais sobre o kernel Linux. Tive a oportunidade de compilar e modificar o kernel, algo que nunca tinha feito antes. Contribuímos com os módulos ad7607 e ad7923 do subsistema IIO, aplicando sugestões de uso do claim_direct_scoped. Um dos nossos patches foi integrado, enquanto o outro necessitou de mais revisões.

### Licenças
A aula sobre licenças foi uma das mais valiosas para mim. Antes, eu tinha pouco conhecimento sobre o assunto. Agora, tenho uma compreensão sólida das diferentes licenças e suas implicações, o que é essencial para qualquer desenvolvedor de software livre.

### KW
Nossa contribuição inicial para o KW foi uma pequena correção em uma variável redefinida desnecessariamente. Este primeiro passo foi rápido e nos preparou para um envolvimento mais profundo com o projeto na segunda parte da disciplina, onde criamos scripts de autocompletar para o KW usando Nushell.


### Contribuições para o Nushell
Adotei o Nushell como meu shell padrão e comecei a explorar suas capacidades. Descobri a coleção nushell/nu_scripts e comecei a usar e contribuir com scripts utilitários. Nosso maior desafio foi criar completions para o KW, o que envolveu bastante aprendizado e experimentação.


### Empacotamento
Participamos de um tutorial com o Joênio sobre empacotamento Debian, e depois tentamos empacotar o Nushell. No entanto, encontramos uma barreira significativa: o grande número de dependências não empacotadas para Debian. Isso nos ensinou sobre as complexidades do empacotamento de software e a importância das dependências bem gerenciadas.

### Conclusão

A disciplina MAC0470 foi uma experiência transformadora. Aprendi sobre kernel, licenças, empacotamento e contribuições práticas para projetos open source. Estou ansiosa para continuar essa jornada em disciplinas futuras e contribuir cada vez mais com o ecossistema de software livre.
